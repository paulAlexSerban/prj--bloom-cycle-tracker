# ADR-005: Tailwind CSS and Radix UI

## Status
Accepted

## Date
2026-02-08

## Context
The app must be accessible and visually consistent with minimal custom styling overhead.

## Decision
Use Tailwind CSS for styling and Radix UI primitives for accessibility-focused components.

## Consequences
- Consistent design system and rapid UI iteration.
- Accessibility benefits from Radix primitives.
- Requires Tailwind configuration and class-based styling.
