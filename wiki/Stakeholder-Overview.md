# Stakeholder Overview Documentation

## Stakeholders

### End Users
**Who:** People who menstruate and want a private, reliable tracking tool.
**Needs:** Easy logging, accurate predictions, symptom insights, privacy assurance, offline access.
**Success Measures:** Trust in privacy, prediction usefulness, daily usability.

### Healthcare Providers (Indirect)
**Who:** Doctors or clinicians reviewing exported user data.
**Needs:** Clear, structured export data to support consultations.
**Success Measures:** Data clarity and usefulness in medical discussions.

### Product Owner / Maintainers
**Who:** Project owner and open-source maintainers.
**Needs:** Sustainable roadmap, clear scope, quality codebase, user trust.
**Success Measures:** Stable releases, positive community feedback, low defect rate.

### Designers / UX Contributors
**Who:** Contributors focused on accessibility and UX.
**Needs:** Consistent component library, accessibility standards, user feedback.
**Success Measures:** High usability scores, accessible UI compliance.

### Privacy/Compliance Reviewers (Advisory)
**Who:** Individuals ensuring data handling respects privacy principles.
**Needs:** Clear data flow and storage policies.
**Success Measures:** No external data transmission by default, transparent user controls.

### Third-Party Calendar Providers (Google Calendar)
**Who:** External service used for optional calendar sync.
**Needs:** OAuth-based access, clear scopes, rate-limit compliance.
**Success Measures:** Reliable sync, minimal data sharing, user-controlled revocation.

## Communication Needs
- Product roadmap and MVP scope.
- Privacy posture and data handling model.
- Known platform limitations (iOS notifications).
- Planned post-MVP enhancements.
